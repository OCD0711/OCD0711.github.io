1. 常用命令

Hexo 的命令极简单, 安装后只需要记住四个常用的即可. 执行命令需要Git当前处于blog文件夹根目录下.

  1. generate 生成静态文件.

    hexo g

  2. server 启动服务器.

    hexo s

  3. deploy 部署网站. 部署网站前, 需要预先生成静态文件.

    hexo d

  4. clean 清除缓存文件 (db.json) 和已生成的静态文件 (public).

    hexo clean

  5. 卸载Hexo.
    npm uninstall hexo-cli -g

4. 设置主题风格

  • 打开 themes/next 下的 _config.yml 文件, 搜索 scheme 关键字, 需用启用的 scheme 前面注释 # 去除即可.

5. 设置菜单项的显示文本和图标

  • 更新说明：NexT.Pisces v5.1.3, 版本更换了修改菜单图标方式. 参看详细信息

  • NexT 使用的是 Font Awesome 提供的图标， Font Awesome 提供了 600+ 的图标，可以满足绝大的多数的场景，同时无须担心在 Retina 屏幕下 图标模糊的问题。

5.1 设置菜单项的显示文本:

  • 打开 themes/next/languages 下的 zh-Hans.yml 文件, 搜索 menu关键字, 修改对应中文或者新增

5.2 设定菜单项的图标

  • 打开 themes/next 下的 _config.yml 文件, 搜索 menu_icons 关键字, 修改对应图标名称或者新增对应 menu 的图标

5.3 设置菜单项对应的文件目录

  • 打开 themes/next 下的 _config.yml 文件, 搜索 menu 关键字, 以 # 注释原有的菜单项, 或者新增新的菜单项

  • 除了 home, archives,/ 后面都需要手动创建这个页面

5.4 创建菜单项对应文件目录, 以分类为例

  • 在终端窗口下, 定位到 Hexo 站点目录下. 使用 hexo new page 新建一个页面, 命名为 categories:

    5.5 编辑刚新建的页面, 设置分类

6. 头像设置

6.1 添加头像

  • 打开 themes/next 下的 _config.yml 文件, 搜索 Sidebar Avatar 关键字, 去掉 avatar 前面的 #

  • 或者使用本地图片, 把图片放入 themes/next/source/images 下, 修改 avatar

    avatar: /images/blogLogo.png

7. 浏览页面的时候显示当前浏览进度

  • 打开 themes/next 下的 _config.yml, 搜索关键字 scrollpercent, 把false改为true
  • 如果想把 top 按钮放在侧边栏, 打开 themes/next 下的 _config.yml, 搜索关键字 b2t, 把 false 改为 true
