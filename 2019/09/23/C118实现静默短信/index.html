C118 osmocombb 实现静默短信

发表于 2019-09-23 | 阅读次数：
本文字数： 7.2k | 阅读时长 ≈ 13 分钟

Osmocom-BB 的 sylvain/testing 分支固件可以配合 OpenBTS,，借助周围信号强度较大的 ARFCN 伪造出一个新的基站信号。不过由于摩托罗拉 C118 的问题，无法实现语音通话功能只可以发送短信

OsmocomBB是国外一个开源项目，是GSM协议栈(Protocols stack)的开源实现，全称是Open source mobile communication Baseband.目的是要实现手机端从物理层(layer1)到layer3的三层实现。

Build.sh 脚本会自动安装所有编译所需要的东西

编译的包会被放在一个新的目录中。这个目录在dev的BUILD中，找到最新的时间戳文件夹。

安装 OpenBTS

注意：这一步需要在安装完GNU Radio之后进行，否则会报找不到UHD包的错误

cd到最新的时间戳文件里

安装编译出来的所有 deb

$ sudo dpkg -i *.deb

这里可能会遇到dependencies的问题，输入下面的命令可以解决:

$ sudo apt-get -f install

启动/停止/重启 OpenBTS

如果上述组件都能被启动，分配到进程号，说明OpenBTS安装成功，这里需要注意一点，前两个进程可以在没有安装完GNU Radio的情况下被启动，openbts和asterisk必须在安装完GNU Radio的情况下才能被启动。是否能被启动的判断依据是是否分配了进程号，如果输入start命令显示的是unknown job，则表示启动失败。

开始执行

到 OpenBTS/apps 目录下, 将 transceiver 重命名为 transceiver.bak 新建脚本文件 transceiver 内容如下

#!/bin/bash exec &lt;your path to osmocom-bb&gt;/src/host/layer23/src/transceiver/transceiver &lt;ARFCN&gt;
